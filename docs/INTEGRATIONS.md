# Integrations

Node Port Stop is designed to fit alongside systems that already handle trust, policy, verification, transport, event history, or operational visibility.

## Integration roles

### Trust and identity systems
These bind node identity and authority.

### Policy systems
These determine what a node is allowed to do and what state it should be in.

### Event and history systems
These record execution, activity, and resulting history around node behavior.

### Verification systems
These evaluate node state, health, contact, and trust-relevant outcomes.

### Transport and exchange systems
These move artifacts, operations, or receipts between nodes and systems.

### Visibility systems
These present fleet state, node health, adjacency, and drift information to operators.

## Why integration matters

Node Port Stop is most valuable when it becomes the connective layer between actors, machines, resources, and system evidence. It is not just a device list. It is the node context layer that makes other systems more attributable and more explainable.
